Exercise 1: Use platform-specific themes (XAM330)

In this exercise, you'll use platform themes to customize our application's appearance.

To complete the exercise, you will need Visual Studio for Windows or macOS with the Xamarin development tools installed. You will also need either an emulator/simulator or a device to run the exercise on. Please see the setup page if you need help installing the Xamarin development environment.

Open the starter solution

Open the starter solution from the Exercise 1 > Start folder in your copy of the cloned or downloaded course materials in either Visual Studio on Windows or Visual Studio for Mac.

Use the iOS Appearance API

  1. Build and run the iOS head-project and inspect the UI.
  2. Open AppDelegate.cs in the iOS project and go to the FinishedLaunching method.
  3. Change the color of the switch in the iOS app by setting the static UISwitch.Appearance.OnTintColor property.
  4. Change the color of the slider's bar by setting the static UISlider.Appearance.MinimumTrackTintColor and UISlider.Appearance.MaximumTrackTintColor properties.
  5. Run the application on iOS and inspect the color changes.

Create a custom Android theme

In this exercise, you'll use the Android material design themes. These are supported in Android 5.0 and above. To see the theme changes, you'll need an emulator or device running Android 5.0 (Lollipop) or newer.

  1. Build and run the Android head-project and inspect the UI.
  2. Open MainActivity.cs in the Android project.
  3. Change the theme to Material.Light in the activity attribute. 
    [Activity(Label = "ControlExplorer", Icon = "@drawable/icon", Theme = "@android:style/Theme.Material.Light", MainLauncher = true, ConfigurationChanges = ConfigChanges.ScreenSize | ConfigChanges.Orientation)]
  4. Run the Android head-project and observe the visual changes, the background should be light or white.
  5. Add a new subfolder in the Android project under the Resources folder named values-v21.
  6. Add a new xml file to the folder named styles.xml.
  7. Add the contents below to create a theme that derives from Material.Light and customizes the accent color. 
    <?xml version="1.0" encoding="utf-8" ?>
<resources>
   <style name="MyCustomTheme" parent="android:Theme.Material.Light">
      <item name="android:colorAccent">#FF9900</item>
   </style>
</resources>
  8. Update the activity attribute in MainActivity.cs to load your custom theme and run the application. 
    [Activity(Label = "ControlExplorer", Icon = "@drawable/icon", Theme = "@style/MyCustomTheme", MainLauncher = true, ConfigurationChanges = ConfigChanges.ScreenSize | ConfigChanges.Orientation)]

Use implicit styles in Windows

  1. If you're using Visual Studio for Windows, build and run the Windows head-project and inspect the UI.
  2. Open App.xaml.
  3. Add an Application.Resources section.
  4. Add an implicit Style targeting Slider.
  5. Add two setters to the Style. One with a Property for Background and the other for Foreground. Select colors for each.

  6. Run the application.

    <Application.Resources>
    <ResourceDictionary>
        <Style TargetType="Slider">
            <Setter Property="Background" Value="Purple" />
            <Setter Property="Foreground" Value="Red" />
        </Style>
    <ResourceDictionary>
</Application.Resources>

Exercise summary

In this exercise you used the platform-specific themes to customize the appearance of controls defined in Xamarin.Forms.

You can view the completed solution in the Exercise 1 > Completed folder of your copy of the cloned or downloaded course materials.
